# Instructions for AI Agents

## Overview

OCI build environment images for building open source RabbitMQ packages (Debian, RPM packages, Windows installer).
For running RabbitMQ, use the Docker community OCI image, [`docker-library/rabbitmq`](https://github.com/docker-library/rabbitmq).

GitHub Actions push them to [`rabbitmqdevenv`](https://hub.docker.com/u/rabbitmqdevenv) on Docker Hub.
The images are then consumed by [`rabbitmq/server-packages`](https://github.com/rabbitmq/server-packages)'s workflows.


## Layout

 * `dockerfiles/<variant>/Dockerfile`: one self-contained Dockerfile per variant
 * `.github/workflows/image-<variant>.yml`: pushes `rabbitmqdevenv/build-env-<variant>:latest`
 * `README.md`: `docker pull` line per variant


## Variants

 * `26.2`, `27.3`: based on Debian Bookworm; Erlang from `deb1.rabbitmq.com` (x86-64 only)
 * `26.2-rocky{8,9,10}`, `27.3-rocky{8,9,10}`: Erlang built from source via [`kerl`](https://github.com/kerl/kerl)

 All images are x86-64 only.


## Pinned Tooling

Every upstream-fetched dependency is pinned by version and SHA256:

 * `ERLANG_VERSION`: used by `apt` version pinning and a version `ARG` for the Rocky Linux variantsa
 * `ELIXIR_VERSION` + `ELIXIR_DOWNLOAD_SHA256`: Elixir compiled from source
 * `REBAR3_VERSION` + `REBAR3_DOWNLOAD_SHA256`: Rocky Linux-specific only; Debian variants use `apt`
 * `OSSLSIGNCODE_VERSION` + `OSSLSIGNCODE_SHA256`: built from source on all images for uniformity (Rocky Linux 8 no longer contains the package)

Bumping a version without updating its SHA256 fails the build at `sha256sum -c -`.
Debian images additionally re-check the installed Erlang version against the `apt`-pinned version.

## Adding a Variant

 1. `dockerfiles/<variant>/Dockerfile`
 2. `.github/workflows/image-<variant>.yml` (copy an existing one, swap name, paths, file, tag)
 3. README `docker pull` line


## Local Builds

`podman build` or `docker build` from `dockerfiles/<variant>`. On aarch64 hosts, Debian variants will need a `--platform linux/amd64` flag.
